## 🔑 Ключевые технические инсайты

### 1. Архитектурный подход: Scriptable Objects (SO)

**Проблема:** Существующая система персонажей и оружия была жестко закодирована

**Решение:** Переход на data-driven архитектуру через Scriptable Objects
- SO хранят конфигурации персонажей и оружия
- UI элементы (индикаторы способностей) привязаны к SO
- Компонент `WeaponSelector` инициализирует UI из SO при старте игры
- Легкое добавление новых персонажей/оружия без изменения кода

**Инсайт:** Студент понимает принципы масштабируемой архитектуры и разделения данных от логики

### 2. Система визуализации способностей

**Реализация:**
- Каждое оружие имеет UI-объект с аниматором состояний (например, "Blaster")
- UI показывает готовность способностей в реальном времени
- Аналогичная система для персонажей ("Character-UI" для "Marine")
- Изображение текущего персонажа также хранится в SO

**Инсайт:** Фокус на UX - игрок всегда видит доступность своих способностей

### 3. AI система на основе FSM (Finite State Machine)

**Архитектура:**
- Состояния хранятся в enum для легкого переключения
- Реализован конечный автомат (FSM) для управления поведением

**Состояния врагов:**
- `Chase` - преследование игрока
- `Strafe` - движение по окружности вокруг игрока
- `Dash` - рывок к игроку

**Типы врагов:**
- `Robot` и `YellowRobot` - используют Chase, Strafe, Dash
- `OrbDroid` (новый) - использует Strafe + стрельба снарядами с интервалами

**Технические детали:**
- Создан отдельный `EnemyWeaponController` (аналог `ProjectileWeaponController`)
- Разделение логики игрока и врагов для избежания конфликтов с системой способностей

**Инсайт:** Понимание паттернов проектирования (State Machine) и модульной архитектуры

### 4. Система обучения игрока (Player Onboarding)

**Философия:** "Чем быстрее и комфортнее пройдет адаптация, тем больше понравится игра"

**Реализованные механизмы:**
1. Описание способностей персонажа и оружия
2. Интерактивный интерфейс с индикаторами готовности
3. **Видео-демонстрация** использования способностей в игровых ситуациях
4. Постепенное погружение в сложность

**Принцип прогрессии:**
- Первая волна врагов - освоение базовых механик
- Последующие волны - усложнение для закрепления навыков
- Игрок учится поведению врагов до того, как столкнется со сложными ситуациями

**Инсайт:** Студент понимает game design принципы и важность player experience

### 5. Система настроек и аудио

**Технология:** PlayerPrefs для сохранения пользовательских настроек
- Данные сохраняются в `AppData/Local/*Название игры*`
- Настройки управления через изменение `InputManager` Unity
- При запуске игры настройки загружаются из сохраненных данных

**Аудио-менеджер:**
- Отдельный GameObject в основной сцене
- Использует встроенный `AudioPlayer` Unity
- Настройка громкости звука
- Музыка взята с NewGrounds (без авторских прав)

**Инсайт:** Внимание к деталям и юридическим аспектам (лицензирование контента)
