**Введение**

Объектно-ориентированное программирование (ООП) - это парадигма программирования, которая позволяет создавать сложные программы, моделирующие реальные объекты и их взаимодействия. В этом разделе мы рассмотрим основы ООП, его отличия от процедурного программирования, а также важность понимания концепций ООП для разработки сложных программ.

**Различия между процедурным и объектно-ориентированным программированием**

До сих пор наши программы состояли из функций, т.е. блоков выражений, которые манипулируют данными. Это называется процедурно-ориентированным стилем программирования. Существует и другой способ организации программ: объединять данные и функционал внутри некоего объекта. Это называется объектно-ориентированной парадигмой программирования.

В большинстве случаев можно ограничиться процедурным программированием, но если решить конкретную задачу требует перехода к техникам объектно-ориентированного программирования. Два основных аспекта объектно-ориентированного программирования – классы и объекты.

**Классы и объекты**

Класс создаёт новый тип, а объекты являются экземплярами класса. Аналогично, когда мы говорим «переменные типа int», это означает, что переменные, которые хранят целочисленные значения, являются экземплярами (объектами) класса int.

Обратите внимание, что даже целые числа рассматриваются как объекты (класса int), в отличие от C++ и Java (до версии 1.5), где целые числа являются примитивами. См. help(int) для более детального описания этого класса.

**Поля и методы**

Объекты могут хранить данные в обычных переменных, которые принадлежат объекту. Переменные, принадлежащие объекту или классу, называют полями. Объект может также обладать функционалом, т.е. иметь функции, принадлежащие классу. Такие функции принято называть методами класса.

Эта терминология важна, так как она помогает нам отличать независимые функции и переменные от тех, что принадлежат классу или объекту. Всё вместе (поля и методы) принято называть атрибутами класса.

**Переменные экземпляра и переменные класса**

Поля могут быть двух типов: они могут принадлежать каждому отдельному экземпляру объекта класса или всему классу. Они называются переменными экземпляра и переменными класса соответственно.

**Основные концепции**

Объектно-ориентированное программирование (ООП) — это парадигма программирования, которая позволяет организовывать код как набор объектов, каждый из которых представляет собой отдельную сущность или класс. В этом разделе мы рассмотрим основные концепции ООП, включая классы и объекты, переменные экземпляра и переменные класса, методы и пространства имён.

**Классы и объекты**

В ООП класс — это шаблон илиblueprint для создания объектов. Класс определяет свойства и поведение объектов, которые будут созданы на основе этого класса. Объект — это экземпляр класса, который имеет собственные значения свойств и может выполнять свои собственные методы.

Класс создаётся с помощью ключевого слова `class`, а затем следует имя класса. Блоки кода внутри класса определяют свойства и поведение объектов этого класса. Например:
```python
# Пример со страницы 110
class Person:
    pass # Пустой блок
```
В этом примере мы создали класс `Person`, который не имеет никаких свойств или методов.

**Переменные экземпляра и переменные класса**

Переменные экземпляра — это свойства объектов, которые принадлежат конкретному объекту. Переменные класса — это свойства, которые принадлежат всем объектам одного класса.

Например, если у нас есть класс `Person`, то переменная `name` будет переменной экземпляра, а переменная `age` будет переменной класса.
```python
# Пример со страницы 112
class Person:
    def __init__(self, name):
        self.name = name # Переменная экземпляра
    age = 0 # Переменная класса

p1 = Person('John')
print(p1.name) # Вывод: John
print(Person.age) # Вывод: 0
```
В этом примере мы создали переменную `name` как переменную экземпляра, а переменную `age` как переменную класса.

**Методы**

Метод — это функция, которая принадлежит классу или объекту. Методы используются для определения поведения объектов и выполнения задач.

Например:
```python
# Пример со страницы 113
class Person:
    def sayHi(self):
        print('Привет! Как дела?')

p = Person()
p.sayHi() # Вывод: Привет! Как дела?
```
В этом примере мы создали метод `sayHi`, который принадлежит классу `Person`. Метод `sayHi` печатает сообщение на экран.

**Метод __init__**

Метод `__init__` — это специальный метод, который вызывается при создании объекта. Этот метод используется для инициализации свойств объектов.

Например:
```python
# Пример со страницы 114
class Person:
    def __init__(self, name):
        self.name = name

p = Person('John')
print(p.name) # Вывод: John
```
В этом примере мы создали метод `__init__`, который принимает аргумент `name` и присваивает его свойству `name` объекта.

**Пространства имён**

Пространство имён — это область, в которой определены переменные и функции. В ООП пространством имён является класс или объект.

Например:
```python
# Пример со страницы 115
class Person:
    name = 'John'

print(Person.name) # Вывод: John
```
В этом примере мы создали переменную `name` в пространстве имён `Person`. Мы можем доступиться к этой переменной через имя класса.

**Рекомендации по написанию читаемого и понятного кода в ООП**

Код, написанный на основе ООП, должен быть читаемым и понятным. Это означает, что он должен иметь четкую структуру, логичную названия переменных и методов, а также должно быть легко понять его функциональность.

* Используйте четкие и короткие названия переменных и методов.
* Используйте пространства имён для организации кода и избежания конфликтов имен.

**Объяснение важности использования self в методах класса**

В ООП методы класса имеют одно отличие от обычных функций: они должны иметь дополнительно имя self, добавляемое к началу списка параметров. Однако, при вызове метода не нужно указывать значение для этого параметра — оно будет автоматически присвоено объекту-экземпляру класса.

Использование self имеет следующие преимущества:

* Упрощает доступ к полям и методам объекта.
* Позволяет использовать одинаковый код для различных объектов.
* Улучшает читаемость и понятность кода.

**Советы по debugging и отладке программ с использованием ООП**

Debugging и отладка программ с использованием ООП могут быть более сложными, чем в случае процедурного программирования. Однако, есть несколько советов, которые можно использовать для упрощения этого процесса:

* Используйте печать переменных и результатов операций для понимания поведения программы.
* Используйте отладчики и инструменты для анализа выполнения программы.
* Используйте методы для группировки логически связанных операций и упрощения отладки.
* Используйте комментарии для объяснения сложных частей кода.
