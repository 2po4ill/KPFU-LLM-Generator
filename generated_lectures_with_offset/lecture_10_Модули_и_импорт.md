Как работает импорт:
В начале мы импортируем модуль sys командой import. Этим мы говорим Python, что хотим использовать этот модуль. Модуль sys содержит функции, относящиеся к интерпретатору Python и его среде, т.е. к системе (system). Когда Python выполняет команду import sys, он ищет модуль sys. В данном случае это один из встроенных модулей, и Python знает, где его искать.

Если бы это был не скомпилированный модуль, т.е. модуль, написанный на Python, тогда интерпретатор Python искал бы его в каталогах, перечисленных в переменной sys.path. Если модуль найден, выполняются команды внутри модуля, и он становится доступным. Обратите внимание, что инициализация происходит только при первом импорте модуля.

Доступ к переменной argv в модуле sys предоставляется при помощи функции sys.argv. Это явно показывает, что это имя является частью модуля sys. Ещё одним преимуществом такого обозначения является то, что имя не конфликтует с именем переменной argv, которая может использоваться в вашей программе.

Переменная sys.argv является списком строк (списки будут детально обсуждаться во второй главе). Она содержит список аргументов командной строки, т.е. аргументов, переданных программе из командной строки. Если вы используете среду разработки для написания и запуска программ, поищите где-нибудь в её меню возможность передавать параметры командной строки.

В нашем примере, когда мы запускаем “python using_sys.py we are arguments”, мы запускаем модуль using_sys.py командой python, а всё, что следует далее – аргументы, передаваемые программе. Python сохраняет аргументы командной строки в переменной sys.argv для дальнейшего использования.

Помните, что имя запускаемого сценария всегда является первым аргументом в списке sys.argv. Так что в приведённом примере 'using_sys.py' будет элементом sys.argv[0], 'we' – sys.argv[1], 'are' – sys.argv[2], а 'arguments' – sys.argv[3]. Помните, что в Python нумерация начинается с 0, а не с 1.

**Основные концепции**

В этом разделе мы рассмотрим основные концепции модулей и импорта в Python. Мы начнем с описания модулей, их назначения и примеров использования.

Модули в Python — это файлы с расширением .py, которые содержат функции, классы или переменные, которые можно использовать в других программах. Модули могут быть написаны на Python или импортировать другие модули. Основная цель модулей — разделить программу на логически связанные части и сделать ее более организованной.

Например, если у вас есть большая программа, которая выполняет различные задачи, вы можете разделить ее на несколько модулей, каждый из которых будет отвечать за конкретную функцию. Это сделает вашу программу более легкой в поддержке и обслуживании.

Модули могут быть импортированы в другие программы с помощью команды `import`. Например:

```python
import math
```

Эта команда импортирует модуль `math`, который содержит функции для математических операций, таких как `sin`, `cos` и `sqrt`.

Модули могут также иметь свои собственные переменные, которые можно использовать в других программах. Например:

```python
import math
print(math.pi)
```

Эта команда выводит значение числа пи, которое определено в модуле `math`.

Процесс импорта модулей включает в себя несколько этапов:

1. Импорт модуля: когда вы используете команду `import`, Python ищет модуль по имени.
2. Поиск модуля: если модуль не найден в текущем каталоге, Python ищет его в других каталогах, перечисленных в переменной `sys.path`.
3. Загрузка модуля: когда модуль найден, он загружается в память и становится доступным для использования.

Переменная `sys.path` содержит список имён каталогов, откуда импортируются модули. Если модуль не найден в текущем каталоге, Python ищет его в других каталогах, перечисленных в переменной `sys.path`.

Например:

```python
import sys
print(sys.path)
```

Эта команда выводит список имён каталогов, откуда импортируются модули.

Байткод-файлы .pyc — это промежуточная форма, в которую Python переводит программу. Эти файлы создаются при компиляции программы и содержат байт-код, который можно использовать для быстрого выполнения программы.

Например:

```python
import mymodule
```

Эта команда импортирует модуль `mymodule`, который был скомпилирован в файл .pyc.

Оператор `from ... import ...` позволяет импортировать конкретные переменные или функции из модуля. Например:

```python
from math import sin, cos
```

Эта команда импортирует функции `sin` и `cos` из модуля `math`.

Имя модуля — это уникальное имя, которое присваивается каждому модулю при его создании. Имя модуля можно использовать для доступа к переменным или функциям модуля.

Например:

```python
import mymodule
print(mymodule.__name__)
```

Эта команда выводит имя модуля `mymodule`.

Создание собственных модулей — это простой процесс. Для этого нужно создать файл с расширением .py и добавить в него функции, классы или переменные.

Например:

```python
# mymodule.py
def sayhi():
    print('Привет! Это говорит мой модуль.')
```

Этот файл является собственным модулем, который можно импортировать в другие программы.

Пакеты — это удобный способ иерархически организовать модули. Пакеты — это каталоги, которые содержат модули и специальный файл `__init__.py`, который показывает Python, что этот каталог является пакетом.

Например:

```python
# world/
#   __init__.py
#   asia/
#       __init__.py
#       india.py
#   africa/
#       __init__.py
#       madagascar.py
```

Этот каталог является пакетом `world`, который содержит модули `asia` и `africa`.

Функция `dir()` позволяет получить список атрибутов объекта. Например:

```python
import sys
print(dir(sys))
```

Эта команда выводит список атрибутов модуля `sys`.

В этом разделе мы рассмотрим практические советы по работе с модулями и импортом в Python.

### Совет по избеганию конфликтов имён при использовании оператора from ... import ...

Оператор `from` позволяет импортировать конкретные элементы из модуля, но он может привести к конфликтам имён. Например, если вы импортируете переменную `argv` из модуля `sys`, она будет иметь тот же именной вид, что и переменная `argv` в вашей программе. Чтобы избежать этого, следует использовать оператор `import` вместо `from`.

Пример:
```
# Плохой пример: импортирование argv из sys с помощью from
from sys import argv

# Хороший пример: импортирование всего модуля sys с помощью import
import sys
```
