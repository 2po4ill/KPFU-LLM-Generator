**Пакеты, модули и структуры данных**

В Python пакеты и модули являются важными концепциями для организации кода. Пакет представляет собой набор связанных между собой модулей, которые могут быть использованы вместе или отдельно. Модуль — это файл с Python-кодом, который может содержать функции, переменные и классы.

Структуры данных являются основным инструментом для хранения и обработки данных в программе. В Python существует четыре основных структуры данных: списки, кортежи, словари и множества. Каждая из этих структур имеет свои особенности и используется для решения конкретных задач.

**Списки**

Список — это изменяемый тип данных, который содержит упорядоченный набор элементов. Списки можно представить как список покупок, где каждый элемент представляет собой отдельную позицию в списке. Списки могут быть созданы с помощью квадратных скобок или функции list.

Списки имеют множество методов, которые можно использовать для их обработки. Например, метод append добавляет новый элемент в конец списка, а метод sort сортирует список по алфавиту или числам.

**Примеры использования списков**

# Пример со страницы 87
shoplist = ['яблоки', 'манго', 'морковь', 'бананы']
print('Я должен сделать ', len(shoplist), 'покупок.')
print('Покупки:', end=' ')
for item in shoplist:
    print(item, end=' ')
print('\nТакже нужно купить риса.')
shoplist.append('рис')
print('Теперь мой список покупок таков:', shoplist)
print('Отсортирую-ка я свой список')
shoplist.sort()
print('Отсортированный список покупок выглядит так:', shoplist)

**Кортежи**

Кортеж — это неизменяемый тип данных, который содержит упорядоченный набор элементов. Кортежи можно представить как кортеж животных в зоопарке, где каждый элемент представляет собой отдельную позицию в кортеже.

Кортежи имеют некоторые особенности, которые отличают их от списков. Например, кортежи не могут быть изменены после их создания, а также они не поддерживают методы, такие как append или sort.

**Списки**

Список — это структура данных, которая содержит упорядоченную последовательность элементов. Это означает, что элементы списка имеют определённый порядок и могут быть доступны по индексу. Списки являются изменяемыми типами данных, что означает, что их можно модифицировать после создания.

Список может содержать любые типы данных, включая строки, целые числа, списки и кортежи. Элементы списка могут быть добавлены или удалены в произвольном порядке.

Списки имеют множество методов, которые можно использовать для их обработки. Например, метод `append()` позволяет добавить элемент в конец списка, а метод `sort()` позволяет отсортировать список по определённому критерию.

Пример со страницы [87]:
```
# Это мой список покупок
shoplist = ['яблоки', 'манго', 'морковь', 'бананы']
print('Я должен сделать ', len(shoplist), 'покупок.')
print('Покупки:', end=' ')
for item in shoplist:
    print(item, end=' ')
print('\nТакже нужно купить риса.')
shoplist.append('рис')
print('Теперь мой список покупок таков:', shoplist)
```
В этом примере мы создали список `shoplist` и добавили в него несколько элементов. Затем мы использовали метод `append()` для добавления нового элемента в конец списка.

**Кортежи**

Кортеж — это структура данных, которая содержит упорядоченную последовательность элементов, но не может быть изменена после создания. Это означает, что кортеж является неизменяемым типом данных.

Кортежи используются для хранения постоянных данных, которые не должны быть модифицированы. Например, кортеж можно использовать для хранения координат точки в пространстве.

Кортежи имеют те же методы, что и списки, но они не могут быть изменены после создания. Пример со страницы [90]:

```
zoo = ('питон', 'слон', 'пингвин') # помните, что скобки не обязательны

```
В этом примере мы создали кортеж `zoo` и добавили его в новый кортеж `new_zoo`.

**Словари**

Словарь — это структура данных, которая содержит набор ключей и значений. Ключи словаря должны быть уникальными, а значения могут быть любыми типами данных.

Словари используются для хранения данных в виде пар ключ-значение. Например, словарь можно использовать для хранения адресной книги.
