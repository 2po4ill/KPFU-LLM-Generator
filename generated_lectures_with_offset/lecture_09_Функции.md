**Введение**

Тема функций является фундаментальной концепцией в программировании, позволяя повторно использовать код и упростить его написание. В этом разделе мы рассмотрим важность функций, их назначение и основные понятия, связанные с ними.

Функции являются блоками кода, которые можно вызывать несколько раз в программе, выполняя определенные действия. Они позволяют повторно использовать код, избегая дублирования его в разных местах программы. Функции также упрощают написание кода, делая его более читаемым и поддерживаемым.

В этом разделе мы рассмотрим следующие ключевые понятия функций:

* Параметры функций: переменные, которые передаются функции при ее вызове.
* Локальные переменные: переменные, объявленные внутри функции, не доступны за пределами функции.
* Зарезервированные слова «global» и «nonlocal»: используемые для определения области видимости переменных.

**Параметры функций**

Функции могут принимать параметры, которые передаются им при вызове. Параметры являются переменными, которые принимаются функцией и используются внутри нее. Они аналогичны переменным, за исключением того, что их значения указываются при вызове функции.

Параметры указываются в скобках при объявлении функции и разделяются запятыми. Аналогично, мы передаем значения параметров, когда вызываем функцию. Обратите внимание на терминологию: имена, указанные в объявлении функции, называются параметрами, а значения, которые передаются функции при ее вызове, — аргументами.

Пример со страницы [68]:
```python
def printMax(a, b):
    if a > b:
        print(a, 'максимально')
    elif a == b:
        print(a, 'равно', b)
    else:
        print(b, 'максимально')
printMax(3, 4) # прямая передача значений
x = 5
y = 7
printMax(x, y) # передача переменных в качестве аргументов
```
В этом примере функция `printMax` принимает два параметра: `a` и `b`. Мы вызываем функцию с прямой передачей значений и с передачей переменных в качестве аргументов.

Параметры указываются в скобках при объявлении функции и разделяются запятыми. Аналогично мы передаём значения, когда вызываем функцию. **Локальные переменные**

При объявлении переменных внутри определения функции они не связаны с другими переменными с таким же именем за пределами функции – т.е. имена переменных являются локальными в функции. Это называется областью видимости переменной. Область видимости всех переменных ограничена блоком, в котором они объявлены, начиная со строки объявления имени.

# Пример со страницы [69]
x = 50
def func(x):
    print('x равен', x)
    x = 2
    print('Замена локального x на', x)
func(x)
print('x по прежнему', x)

В этом примере мы определили функцию с именем func, которая использует параметр x. Мы находимся внутри этой функции и выводим значение x. Далее мы назначаем x значение 2. Имя x локально для нашей функции. Поэтому когда мы заменим значение x в функции, значение x, объявленное в основном блоке, останется незатронутым.

Последний вызов функции print выводит значение x, указанное в основном блоке, подтверждая таким образом, что оно не изменилось при локальном присваивании значения в вызванной функции.

**Зарезервированное слово «global»**

Чтобы присвоить некоторое значение переменной, определённой на высшем уровне программы (т.е. не в какой-либо области видимости, как то функции или классов), необходимо указать Python, что это имя является локально, а глобально (global). Сделаем это с помощью зарезервированного слова global. Без применения зарезервированного слова global невозможно присвоить значение переменной, определённой за пределами функции.

# Пример со страницы [70]
x = 50
def func():
    global x
    print('x равно', x)
    x = 2
    print('Заменяем глобальное значение x на', x)
func()
print('Значение x составляет', x)

В этом примере мы определили функцию с именем func, которая использует зарезервированное слово global для объявления переменной x как глобальной. Когда мы присваиваем значение x в функции, это изменение отражается на значении переменной x в основном блоке программы.

Используя одно зарезервированное слово global, можно объявить сразу несколько переменных: global x, y, z.

**Зарезервированное слово «nonlocal»**

Мы видели, как получать доступ к переменным в локальной и глобальной области видимости. Есть ещё одна область видимости, называемая «нелокальной» (nonlocal) областью видимости, которая представляет собой что-то среднее между первыми двумя. Нелокальные области видимости встречаются, когда вы определяете функцию внутри другой функции.

# Пример со страницы [71]
def func_outer():
    x = 2
    print('x равно', x)
    def func_inner():
        nonlocal x
        x = 5
    func_inner()
    print('Локальное x сменилось на', x)
func_outer()

В этом примере мы определили функцию с именем func_outer, которая использует зарезервированное слово nonlocal для объявления переменной x как нелокальной. Когда мы присваиваем значение x в функции func_inner, это изменение отражается на значении переменной x в области видимости func_outer.

**Значения аргументов по умолчанию**

Зачастую часть параметров функций могут быть необязательными, и для них будут использоваться некоторые заданные значения по умолчанию, если пользователь не укажет свои собственные. Это можно достичь с помощью значений аргументов по умолчанию. Их можно указать, добавив к имени параметра в определении функции оператор присваивания (=) со следующим значением.

# Пример со страницы [72]
def say(message, times = 1):
    print(message * times)
say('Привет')
say('Мир', 5)

В этом примере мы определили функцию с именем say, которая использует значение аргумента по умолчанию для параметра times. Если мы не указываем значения, строка выводится один раз. Мы достигаем этого, указывая значение аргумента по умолчанию, равное 1 для параметра times.

При первом вызове say мы указываем только строку, и функция выводит её один раз. При втором вызове say мы указываем также и аргумент 5, обозначая таким образом, что мы хотим сказать фразу 5 раз.

**Ключевые аргументы**

В этом случае для передачи аргументов функции используется имя (ключ) вместо позиции (как обычно).

Есть два преимущества такого подхода: во-первых, использование функции становится легче, поскольку не требуется отслеживать порядок аргументов; во-вторых, можно задавать значения только некоторым избранным аргументам, при условии, что остальные параметры имеют значения аргумента по умолчанию.

**Пример 5: Использование переменного числа параметров**

На странице [74] учебника написано, что можно использовать звездочки (*) для определения функций с переменным числом параметров. Для этого можно указать значение по умолчанию для первого параметра и использовать звездочки для остальных параметров.

В этом разделе мы рассмотрим практические советы по написанию эффективного и читаемого кода с использованием функций. Мы также обсудим важность тестирования и отладки кода.

### Рекомендации по написанию эффективного и читаемого кода

1.  **Используйте meaningful названия функций**: Имя функции должно точно отражать ее поведение. Это поможет другим разработчикам понять, что делает функция.
2.  **Следуйте принципам SOLID**: SOLID — это набор принципов для написания качественного кода. Он включает в себя следующие принципы:
    *   **Single responsibility principle (SRP)**: Каждая функция должна иметь одну и только одну ответственность.
    *   **Open/closed principle (OCP)**: Функции должны быть открыты для расширения, но закрыты для модификации.
    *   **Liskov substitution principle (LSP)**: Функции должны быть заменимыми на более специализированные функции.
    *   **Interface segregation principle (ISP)**: Функции должны иметь четко определенные интерфейсы.
    *   **Dependency inversion principle (DIP)**: Функции должны зависеть от абстракций, а не от конкретных реализаций.
3.  **Используйте функциональные программирование**: Функциональное программирование — это стиль программирования, который фокусируется на использовании функций для решения проблемы, а не на изменении состояния.
4.  **Используйте тестирование и отладку**: Тестирование и отладка являются важными шагами в разработке программного обеспечения. Они помогают выявить ошибки и улучшить качество кода.

1.  **Используйте meaningful названия переменных**: Имя переменной должно точно отражать ее значение.

**Резюме основных концепций**

В лекции были рассмотрены основные концепции функций в Python. Мы узнали, что функция - это блок кода, который можно вызывать несколько раз из разных мест программы. Функции могут принимать параметры, которые передаются ей при вызове, и они могут возвращать значения.

**Параметры функций**

Параметры функций - это переменные, которые принимаются функцией при ее вызове. Они указываются в скобках при объявлении функции и разделяются запятыми. Аналогично, при вызове функции передаются значения параметров, разделенные запятыми.

**Локальные переменные**

Локальные переменные - это переменные, которые объявляются внутри функции и имеют область видимости только в пределах этой функции. Они не связаны с переменными с тем же именем за пределами функции.

**Зарезервированное слово «global»**

Зарезервированное слово «global» используется для объявления переменной как глобальной, т.е. доступной из любой части программы.

**Зарезервированное слово «nonlocal»**

Зарезервированное слово «nonlocal» используется для объявления переменной как не локальной, т.е. доступной из функций, которые находятся внутри другой функции.

**Значения аргументов по умолчанию**

Значения аргументов по умолчанию - это значения, которые используются в качестве значений параметров, если они не указаны
