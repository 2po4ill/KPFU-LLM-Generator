**7.2 Литеральные константы**

Примером литеральной константы может быть число, например, 5, 1.23, 9.25e-3 или что-то вроде 'Это строка' или "It's a string!". Они называются литеральными, потому что они «буквальны» — вы используете их значение буквально.

Число 2 всегда представляет собой себя в виде целого числа и не может быть изменено. Поэтому все это называется литеральной константами.

**7.3 Числа**

Числа в Python бывают трех типов: целые, с плавающей точкой (или «плавающие» для краткости) и комплексные.

* Примером целого числа может служить 2.
* Примерами чисел с плавающей точкой (или «плавающих» для краткости) могут быть 3.23 и 52.3E-4. Обозначение E показывает степень числа 10. В данном случае 52.3E-4 означает 52,3 × 10^(-4).
* Примеры комплексных чисел: (-5+4j) и (2.3 - 4.6j)

Замечание для опытных программистов:

Нет отдельного типа ‘long int’ (длинное целое). Целые числа по умолчанию могут быть произвольной длины.

**7.4 Строки**

Строка — это последовательность символов. Чаще всего строки — это просто некоторые наборы слов.

Слова могут быть как на английском языке, так и на любом другом, поддерживаемом стандартом Unicode, что означает почти на любом языке мира.

Замечание для опытных программистов:

В Python 3 нет ASCII-строк, потому что Unicode является надмножеством (включает в себя) ASCII. Если необходимо получить строку строго в кодировке ASCII, используйте str.encode("ascii"). Подробнее см. обсуждение этого вопроса на StackOverflow.

**7.4.1 Одинарные кавычки**

Строку можно указать, используя одинарные кавычки, как например, 'Фраза в кавычках'. Всё пробелы и табуляция сохранятся, как есть.

**7.4.2 Двойные кавычки**

Стоки в двойных кавычках работают точно так же, как и в одинарных. Например, "What's your name?".

**7.4.3 Тройные кавычки**

Можно указывать «многострочные» строки с помощью использования тройных кавычек (""" или '''). В пределах тройных кавычек можно свободно использовать двойные или тройные кавычки.


**7.4.6 Метод format**

Иногда бывает необходимо составить строку на основе каких-либо данных. Вот здесь-то и приходится использовать метод format().

Сохраните следующий файл в str_format.py:
```python
age = 26
name = 'Swaroop'
print('Возраст {0} -- {1} лет.'.format(name, age))
print('Почему {0} забавляется с этим Python?'.format(name))
```

Вывод:
```
$ python str_format.py
Возраст Swaroop -- 26 лет.
Почему Swaroop забавляется с этим Python?
```

Как это работает:

В строку могут быть включены определённые обозначения, а впоследствии может быть вызван метод format для замещения этих обозначений соответствующими аргументами.

Взгляните на первый случай применения обозначений, где мы пишем {0}, и это соответствует переменной name, являющейся первым аргументом метода format. Аналогично, второе обозначение {1} соответствует переменной age, являющейся вторым аргументом метода format. Заметьте, что Python начинает отсчёт с 0, поэтому первая позиция — номер 0, вторая — номер 1 и т. д.

Заметьте, мы могли бы добиться того же результата и объединением строк: 'Возраст' + name + ' -- ' + str(age) + ' лет.', но вы сами видите, как это некрасиво, и как легко в таком случае допустить ошибку. Во-вторых, преобразование в строку делается методом format автоматически, в отличие от явного преобразования в нашем примере. В-третьих, используя метод format, мы можем изменить сообщение, не затрагивая используемых переменных, и наоборот.

На всякий случай имейте в виду, что цифры здесь не обязательны. Можно было бы просто написать:
```python
age = 26
name = 'Swaroop'
print('Возраст {} -- {} лет.'.format(name, age))
print('Почему {} забавляется с этим Python?'.format(name))
```

и получить тот же результат.

Метод format Python помещает значение каждого аргумента в обозначенное место. Могут быть и более детальные обозначения, как то:

* десятичное число (.) с точностью в 3 знака для плавающих:
```python
'{0:.3}'.format(1/3)
```
* заполнить подчёркиваниями (_) с центровкой текста (^) по ширине 11:
```python
'{0:_^11}'.format('hello')
```
* по ключевым словам:
```python
'{name} написал {book}'.format(name='Swaroop', book='A Byte of Python')
```

Детально такие обозначения форматов описаны в Предложении по расширению Python PEP3101.

### Обработка исключений

Обработка исключений в Python выполняется при помощи оператора try..except. Этот оператор позволяет помещать команды, которые могут вызвать исключения/ошибки, внутрь блока try, а затем поместить обработчики соответствующих ошибок/исключений в блок except.

**Пример со страницы 128**
```python
try:
    text = input('Введите что-нибудь --> ')
except EOFError:
    print('Ну зачем вы сделали мне EOF?')
except KeyboardInterrupt:
    print('Вы отменили операцию.')
else:
    print('Вы ввели {0}'.format(text))
```
В этом примере мы используем оператор try..except для обработки исключений. Мы помещаем команды, которые могут вызвать исключения/ошибки, внутрь блока try, а затем поместим обработчики соответствующих ошибок/исключений в блок except.
