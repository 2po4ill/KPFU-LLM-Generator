**ОСНОВЫ ОБЪЕКТНО-ОРИЕНТИРОВАННОГО ПРОГРАММИРОВАНИЯ**

**ВВЕДЕНИЕ**

Объектно-ориентированное программирование (ООП) - это парадигма программирования, которая позволяет создавать сложные системы путем организации кода в виде объектов. ООП широко используется в современном программировании и является фундаментальной частью многих языков программирования.

Актуальность темы заключается в том, что [ТРЕБУЕТ ПРОВЕРКИ (уверенность: 7%): ООП позволяет решить проблемы, связанные с развитием сложных систем, а также улучшить читаемость и поддержку кода.] Цели и задачи лекции - детально изучить основные концепции ООП, их принципы и применение на практике.

Связь с предыдущими темами заключается в том, что понимание ООП требует базового знания языка программирования и его синтаксиса. В частности, для этого лекции необходимо иметь базовые знания Python.

Краткий обзор содержания:

* Основные концепции ООП: объекты, классы, наследование, полиморфизм.
* Принципы работы с ООП: создание и использование объектов, наследование и полиморфизм.
* Практические примеры кода на Python.

**ОСНОВНАЯ ЧАСТЬ**

**1. Объекты и классы**

В ООП объект - это экземпляр класса, который представляет собой набор свойств (атрибутов) и поведения (методов). Класс - это шаблон илиblueprint для создания объектов.

Пример:
```python
class Автомобиль:
    def __init__(self, марка, модель):
        self.марка = марка
        self.модель = модель

автомобиль1 = Автомобиль('Тойота', 'Королла')
print(автомобиль1.марка)  # Тойота
```
В этом примере `Автомобиль` - это класс, а `автомобиль1` - объект этого класса.

**2. Наследование**

[ТРЕБУЕТ ПРОВЕРКИ (уверенность: 5%): Наследование позволяет создать новый класс на основе существующего класса.] [ТРЕБУЕТ ПРОВЕРКИ (уверенность: 3%): Новый класс наследует свойства и поведение родительского класса.]

Пример:
```python
class Электромобиль(Автомобиль):
    def __init__(self, марка, модель, батарея):
        super().__init__(марка, модель)
        self.батарея = батарея

электромобиль1 = Электромобиль('Тойота', 'Королла', '100 кВт')
print(электромобиль1.марка)  # Тойота
```
В этом примере `Электромобиль` - это класс, который наследует свойства и поведение от `Автомобиль`.

**3. Полиморфизм**

[ТРЕБУЕТ ПРОВЕРКИ (уверенность: 7%): Полиморфизм позволяет создать методы с одинаковым названием в разных классах.]

Пример:
```python
class Автомобиль:
    def движение(self):
        print('Двигает автомобиль')

class Электромобиль(Автомобиль):
    def движение(self):
        print('Двигает электромобиль')

автомобиль1 = Автомобиль()
электромобиль1 = Электромобиль()

автомобиль1.движение()  # Двигает автомобиль
электромобиль1.движение()  # Двигает электромобиль
```
В этом примере `движение` - это метод, который определен в классах `Автомобиль` и `Электромобиль`.

**4. Принципы работы с ООП**

* Создание объектов: создать экземпляр класса.
* Наследование: создать новый класс на основе существующего класса.
* Полиморфизм: создать методы с одинаковым названием в разных классах.

**5. Сравнение различных подходов**

ООП имеет ряд преимуществ, таких как:

* Улучшение читаемости и поддержки кода.
* Решение проблем, связанных с развитием сложных систем.
* Упрощение создания новых классов и объектов.

Недостатками ООП являются:

* Затраты времени на изучение и применение концепций ООП.
* Невозможность использования ООП в некоторых случаях.
